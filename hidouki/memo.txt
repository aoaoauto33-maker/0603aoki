<promise>
非同期処理がより簡単で見やすくなる書き方
これだけ覚えれば前のやつはいらないかもー

new Promise(
    同期処理
).then(
    非同期処理(resolveを待つ)
).catch(
    非同期処理(rejectを待つ)
).finally(
    非同期処理(then,またはcatchを待つ)
);

<Promise.allSettled>
Promise.allとPromise.allSettledの違いは何？
→Promise.allは一つでも失敗したら全体が失敗
→Promise.allSettledは成功も失敗も全部もつ





<MacrotasksとMicrotasks part1>
マクロタスク...タスクキューと呼ばれているもの
setTimeoutはマイクロタスクにタスクを登録している
順番が回ってきたら１つずつタスクを実行

マイクロタスク...タスクキューとは別で存在する非同期処理の待ち行列、ジョブキューという
promiseで登録した非同期処理が含まれる
順番が回ってきた全てのジョブを実行


マクロタスクの方に先にタスクキューされていても、マイクロタスクが優先される
つまり、setTimeoutよりPromiseを優先

上と下は一緒、書き方が違うだけ
queueMicrotask(function job4() {
    console.log('job4');
});

const promise = Promise.resolve();
promise.then(function job4() {
    console.log('job4');
});



<AwaitとAsync>
Promiseをさらに直感的に記述できるようにしたもの
Promise=必ず戻すからね、という意味でつけられた
.thenの理解はAwaitとAsyncを使うのに必要だった

Async
この関数は非同期だよと宣言するもの、関数より前につける
必ずPromiseを返却する関数の宣言を行う

Await
待つ
処理が終わるまで他の処理に邪魔されずに待てる
Promiseを返却する関数の非同期処理が完了するまで待ちたいときに使う


async functionと.then()はイコールではない
awaitの続きの処理は.then()に近い動きをする
